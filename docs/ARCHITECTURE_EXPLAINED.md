## 🔄 Полный цикл работы

### Шаг 1: Solid Queue запускает Job (каждые 5 минут)

```yaml
# config/recurring.yml
fetch_ruby_vacancies:
  class: FetchVacanciesJob
  args: [:ruby]
  schedule: every 5 minutes
```

**Что происходит:**
- Rails смотрит в `recurring.yml`
- Каждые 5 минут создаёт задачу: `FetchVacanciesJob.perform_later(:ruby)`
- Solid Queue берёт задачу из очереди и выполняет

---

### Шаг 2: FetchVacanciesJob запускает цепочку

```ruby
class FetchVacanciesJob
  def perform(language)
    # 1. Парсим
    raw = HhScraper.new(language: language).fetch_for_query
    
    # 2. Фильтруем
    filtered = VacancyFilter.new(raw, language: language).call
    
    # 3. Сохраняем
    imported = VacancyImporter.new.import(filtered)
    
    # 4. Рассылаем
    TelegramNotifier.new(language: language).send_vacancies(imported) if imported.any?
  end
end
```

---

### Шаг 3: HhScraper идёт на HH.ru

```
Ты: language = :ruby
         ↓
HhScraper:
  query = "Ruby" (из маппинга)
         ↓
GET https://api.hh.ru/vacancies?text=Ruby&per_page=100
         ↓
Ответ HH.ru:
{
  "items": [
    {"id": "123", "name": "Ruby Dev", ...},
    {"id": "456", "name": "Rails Dev", ...}
  ],
  "pages": 5,
  "found": 500
}
         ↓
Возвращает: массив из 500 вакансий (сырой JSON)
```

---

### Шаг 4: VacancyFilter фильтрует

```
Вход: 500 вакансий (все что нашёл HH)
         ↓
Фильтр 1: Дата < 14 дней?
  - "Ruby Dev" (1 день) → ✅
  - "Ruby Dev" (30 дней) → ❌
         ↓
Фильтр 2: Ключевые слова в названии?
  - "Ruby Developer" → ✅ (есть "ruby")
  - "HR Manager" → ❌ (нет "ruby")
         ↓
Фильтр 3: Нет стоп-слов?
  - "Ruby Developer" → ✅
  - "Ruby Intern" → ❌ (intern в стоп-словах)
         ↓
Выход: 50 вакансий
```

---

### Шаг 5: VacancyImporter сохраняет в БД

```
Вход: 50 отфильтрованных вакансий
         ↓
Для каждой:
  if Vacancy.exists?(hh_id: data['id'])
    skip  # Уже есть в БД
  else
    Vacancy.create!(...)  # Сохраняем
  end
         ↓
Из 50:
  - 30 уже были (дубликаты)
  - 20 новых
         ↓
Выход: 20 сохранённых вакансий
```

---

### Шаг 6: TelegramNotifier рассылает

```
Вход: 20 новых вакансий для :ruby
         ↓
1. Найти подписчиков:
   Subscription.where(language: :ruby, active: true)
   
   Находит:
   - @vladimir (telegram_id: 12345)
   - @petr (telegram_id: 99999)
         ↓
2. Для каждого подписчика:
   Сформировать сообщение:
   "💎 Новые вакансии (20)
   
   1️⃣ Ruby Developer
      🏢 ООО Ромашка · Москва
      💰 от 150 000 руб.
      [👉 Открыть](ссылка)
   
   2️⃣ Rails Developer
      ...
   
   (показываем первые 5)
   "
         ↓
3. Отправить через Telegram Bot API:
   @bot.api.send_message(
     chat_id: 12345,
     text: "💎 Новые вакансии...",
     parse_mode: "Markdown"
   )
         ↓
Результат:
- @vladimir получил сообщение ✅
- @petr получил сообщение ✅
```

## ✅ Главный вывод

1. **HhScraper** — ходит на HH.ru, забирает сырые данные
2. **VacancyFilter** — отбирает релевантные (дата + keywords)
3. **VacancyImporter** — сохраняет в БД (копий не создаёт)
4. **TelegramNotifier** — берёт из БД вакансии + подписчиков → отправляет в Telegram
5. **FetchVacanciesJob** — запускает всю цепочку каждые 5 минут

**БД растёт** (~20 вакансий × 12 раз в час × 24 часа = ~5760 в день)  
**Очистка** — удалять вакансии старше 30 дней (чтобы не раздувать БД)
